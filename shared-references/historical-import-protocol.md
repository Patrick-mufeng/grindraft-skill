# 历史文章导入协议 / Historical Import Protocol

> 用户在 grindraft-init 时提供已发布的历史文章，系统做风格提取 + 基准设定 + 可选逆向打分拟合初始权重。
> **核心约束**：历史文章已发布，用户一定看过数据——所有打分必须标注 `retrospective`，不能冒充 blind prediction。

## 核心规则

1. **历史文章永远不是 blind prediction**——用户已经知道阅读量，打分是 retrospective
2. **retrospective 分数用于拟合初始权重，不用于验证预测精度**
3. **calibration_samples 只计 blind prediction + 复盘完成的文章**——历史文章不计入
4. **风格提取是导入的最大价值**——用户已有的写作偏好一次性沉淀到 style_guide.md
5. **基准值设定是第二大价值**——复盘时的"vs 基准"从此有参照物

## 导入流程总览

```
[用户：我有 N 篇历史文章]
  ↓
Phase 3a: 收集文章数据（每篇：标题 + 正文 + 发布日期 + 核心指标）
  ↓
Phase 3b: 落盘到 articles/ 目录
  ↓
Phase 3c: 风格分析（逆向沉淀 → style_guide.md）
  ↓
Phase 3d: 逆向打分（7 维 retrospective scoring）
  ↓
Phase 3e: 拟合初始权重（composite 排序 vs 实际阅读量排序）
  ↓
Phase 3f: 设定基准值（平均分享率/收藏率/留言率 → benchmark.md）
  ↓
Phase 3g: 更新 state（historical_samples / rubric_version / baseline_metrics）
  ↓
Phase 3h: 自检 — 逐项重读验证全部写入
```

## Phase 3a: 收集文章数据

### 最少需要（每篇）

| 字段 | 必需？ | 说明 |
|------|--------|------|
| 标题 | ✅ 必需 | 文章标题 |
| 正文 | ✅ 必需 | 全文内容 |
| 发布日期 | ✅ 必需 | YYYY-MM-DD |
| 阅读量 | ✅ 必需 | 用于拟合权重 |
| 分享数 | 推荐 | 用于计算分享率基准 |
| 收藏数 | 推荐 | 用于计算收藏率基准 |

### 可选（有就更好）

| 字段 | 说明 |
|------|------|
| 在看数 | 用于计算在看率基准 |
| 留言数 | 用于计算留言率基准 |
| 阅读完成率 | 用于验证 NR 维度 |
| 留言摘要 | 用于 persona 分析 |

### 收集方式

询问用户：

```
你有几篇历史文章？每篇我需要以下信息：

最少（每篇都要）：
  标题：_______
  正文：_______（可以直接粘贴全文，或告诉我文件路径）
  发布日期：_______
  阅读量：_______

推荐（有就填）：
  分享数：_______
  收藏数：_______

可选：
  在看数：_______
  留言数：_______
  阅读完成率：_______%
  印象深刻的留言：_______

---

你可以：
A) 一篇一篇给我（我逐篇处理）
B) 一次性全给我（按上面的格式，每篇用 --- 分隔）
C) 告诉我在哪个文件夹，我去读
```

### 最少篇数

- **≥5 篇**：可以做风格提取 + 基准设定 + 权重拟合（完整功能）
- **3-4 篇**：可以做风格提取 + 基准设定（权重保持 v0 等权）
- **1-2 篇**：只能做风格提取（基准值和权重拟合样本不足，保持 v0 等权）

> **1-2 篇时的引导**：如果用户只有 1-2 篇历史文章，主动提示：
> "只有 1-2 篇的话，风格分析可能不太准（样本太少），基准值也算不出来。要不要试试找更多？如果暂时没有更多也没关系——等你写完前 5 篇新文章后，沉淀机制会自动积累你的风格偏好，基准值也会从复盘中自然生长出来。"

## Phase 3c: 风格分析（逆向沉淀）

通读所有历史文章，提取用户写作特征。**与正常沉淀机制不同——这里没有 AI 初稿做对比，直接从用户原文中识别 pattern。**

### 分析维度

| 维度 | 分析什么 | 写入 style_guide.md 的段 |
|------|---------|------------------------|
| **禁用词检测** | 用户从不使用的 inflated 词 | `## 绝对禁区` |
| **口语化特征** | 用户常用的过渡词、口头禅、句式 | `## 推荐口语化词组` |
| **结构偏好** | 段落长度分布、小标题使用频率、开头方式、结尾方式 | `## 风格参数` 和 `## 我的写作人格` |
| **文章原型** | 用户最常写的文章类型 | `## 我的文章原型` |
| **语气特征** | 第一人称使用频率、自嘲/幽默程度、立场鲜明度 | `## 我的写作人格` |

### 分析方法

```
对每篇文章：
  1. 统计段落长度分布（中位数、短段比例、长段比例）
  2. 提取口语化过渡词（正则匹配："说实话""说白了""你想想"等）
  3. 检测开头模式（场景切入 vs 宏大叙事 vs 提问开头）
  4. 检测结尾模式（回环 vs 行动呼吁 vs 留白 vs 口号）
  5. 检测小标题使用频率和风格
  6. 检测加粗使用频率
  7. 提取高频但非 AI 套话的独特表达

跨文章汇总：
  - 出现 ≥2 篇的 pattern → 写入正式规则
  - 只出现 1 篇的 pattern → 标"待验证"
```

### 写入 style_guide.md 的格式

在 style_guide.md 中追加一个 `## 历史文章分析（初始化导入）` 段，记录分析来源：

```markdown
## 历史文章分析（初始化导入）

> 以下 pattern 从你的 {N} 篇历史文章中提取。
> 标记"待验证"的等你下一篇新文章确认后升级为正式规则。

### 从历史文章中识别的风格特征

| 特征 | 出现篇数 | 置信度 | 状态 |
|------|---------|--------|------|
| 段落偏短（中位数 2.8 句） | 6/8 | ⭐⭐⭐ | 已写入风格参数 |
| 从不用"高效""赋能" | 8/8 | ⭐⭐⭐⭐ | 已写入绝对禁区 |
| 常用"说实话"开头 | 5/8 | ⭐⭐⭐ | 已写入推荐口语化词组 |
| 结尾偏好留白 | 4/8 | ⭐⭐ | 待验证 |
```

然后更新 style_guide.md 各段的具体内容。

### 纪律

- **不沉淀一次性内容**（某篇文章特有的术语、案例名）
- **不沉淀格式性特征**（中英文空格等——由 humanize L1 处理）
- **标注来源**：每条从历史文章提取的规则标注 `（来源：历史文章分析，{N}/{M} 篇）`
- **与正常沉淀机制的关系**：历史分析是一次性的初始化。之后正常沉淀机制继续运行，逐步覆盖和细化。

## Phase 3d: 逆向打分（Retrospective Scoring）

对每篇历史文章做 7 维打分。**必须标注 retrospective。**

### 打分流程

1. 读文章正文
2. 按当前 rubric（v0 等权）逐维打分
3. 写 prediction.md

### prediction.md 格式（历史文章版）

```markdown
# Retrospective 分析：{标题}

**Article ID**: {sha256[:12]}
**⚠️ RETROSPECTIVE — NOT A BLIND PREDICTION**
**导入日期**：YYYY-MM-DD
**发布日期**：YYYY-MM-DD
**Scored By**：claude（retrospective）
**数据状态**：用户已在导入时提供阅读量等数据

---

## Retrospective 打分

> ⚠️ 以下打分是在已知阅读量={N}的情况下进行的 retrospective 分析。
> 不用于预测精度验证，仅用于初始权重拟合。

### 7 维打分

| 维度 | 分 | 理由 |
|------|-----|------|
| HK 标题钩力 | 1-5 | 一句话（引用标题原文） |
| NR 叙事牵引力 | 1-5 | 一句话（引用正文具体段落） |
| QA 金句锚点 | 1-5 | 一句话（引用具体金句） |
| UT 实用密度 | 1-5 | 一句话 |
| DT 思辨激发度 | 1-5 | 一句话 |
| EP 情绪峰值 | 1-5 | 一句话 |
| SC 结构闭环 | 1-5 | 一句话 |

→ composite ≈ X.XX（等权）

### 实绩数据

| 指标 | 数值 |
|------|------|
| 阅读量 | {N} |
| 分享数 | {N} |
| 收藏数 | {N} |

### 已知数据下的观察

- [如果某维度分高但数据低 → 标注可能被高估]
- [如果某维度分低但数据高 → 标注可能被低估]
- [如果 composite 排序与阅读量排序一致 → 标注]
```

> ⚠️ **retrospective 观察不入 rubric_notes.md**：这里的观察是基于已知数据的马后炮分析，不是 blind 复盘产生的观察。它们**只写在 prediction.md 的"已知数据下的观察"段**，不写入 rubric_notes.md 的"观察记录"段。rubric_notes.md 只接收 blind prediction → 复盘 → 验证/推翻流程产生的观察。

## Phase 3e: 拟合初始权重

**仅在 historical_samples ≥ 5 时执行。**

### 拟合方法

```
1. 取所有历史文章的 composite（等权 v0）和实际阅读量
2. 按阅读量排序 → 得到"真实排序"
3. 按 composite 排序 → 得到"预测排序"
4. 计算 rank correlation（Spearman）
   - ≥4/5 一致 → v0 等权已足够，保持 v0
   - <4/5 一致 → 尝试调整权重
```

### 权重调整方法（简化回归）

```
对每个维度：
  1. 计算该维度分数与阅读量的 Spearman 相关系数
  2. 相关系数 > 0.3 → 该维度有预测力，权重上调
  3. 相关系数 < -0.1 → 该维度可能反向，权重下调
  4. 相关系数 ≈ 0 → 该维度无预测力，保持默认

调整规则：
  - 强正相关（r > 0.5）→ 权重 ×2.0
  - 中等正相关（r > 0.3）→ 权重 ×1.5
  - 弱相关（-0.1 < r < 0.3）→ 权重不变
  - 负相关（r < -0.1）→ 权重 ×0.5（标记"待验证"）

调整后用新权重重算 composite，再次检查排序一致性。
```

### 拟合结果处理

```
if 调整后 rank correlation ≥ 4/5:
    rubric_version = "v1"
    写入 rubric_notes.md 的"当前公式"段
    标注 "v1 — 从 {N} 篇历史文章拟合"
else:
    保持 v0 等权
    在 rubric_notes.md 的"观察记录"段记录偏差方向
    标注 "v0 等权 — 历史数据不足以拟合可靠权重"
```

### 重要约束

- **拟合的权重只是起点**——后续 bump 流程（全量重打 + 跨模型审）才是正式的权重升级
- **拟合权重不替代 bump**——它只是让 v1 比纯先验猜测更接近用户真实情况
- **如果拟合失败（<4/5 一致）**——诚实保持 v0，不强行拟合

## Phase 3f: 设定基准值

计算用户历史文章的基准指标，写入 benchmark.md。

### 计算方法

```
分享率基准 = 所有历史文章分享数的中位数 / 阅读量的中位数
收藏率基准 = 所有历史文章收藏数的中位数 / 阅读量的中位数
留言率基准 = 所有历史文章留言数的中位数 / 阅读量的中位数
在看率基准 = 所有历史文章在看数的中位数 / 阅读量的中位数
平均阅读量 = 所有历史文章阅读量的中位数
```

> 用中位数而非均值——避免单篇爆款扭曲基准。

### 数据缺失时的降级方案

如果用户只提供了阅读量（没提供分享数/收藏数等）：

```
可以计算的：
  - 平均阅读量 ✅（只要有阅读量就能算）

无法计算的（标"—"）：
  - 分享率 —（没有分享数）
  - 收藏率 —（没有收藏数）
  - 留言率 —（没有留言数）
  - 在看率 —（没有在看数）
```

> **不强行估算**——没有数据就诚实标注"—"，不给虚假基准值。这些指标等到后续新文章的复盘中逐步积累。

### benchmark.md 格式

```markdown
# 基准数据 / Benchmark

> 从 {N} 篇历史文章计算。用于复盘时"vs 基准"对比。
> 最后更新：YYYY-MM-DD

## 我的基准指标

| 指标 | 基准值 | 可用？ |
|------|--------|--------|
| 平均阅读量 | {N} | ✅ |
| 分享率 | {X.X}% 或 — | {'✅' if data else '— 无数据'} |
| 收藏率 | {X.X}% 或 — | {'✅' if data else '— 无数据'} |
| 留言率 | {X.X}% 或 — | {'✅' if data else '— 无数据'} |
| 在看率 | {X.X}% 或 — | {'✅' if data else '— 无数据'} |

> 标注"—"的指标因历史数据中缺少对应数字无法计算。

## 历史文章列表

| # | 标题 | 日期 | 阅读量 | 分享率 | 收藏率 |
|---|------|------|--------|--------|--------|
| 1 | {标题} | {日期} | {N} | {X.X}% | {X.X}% |
| 2 | ... | ... | ... | ... | ... |

## 对标账号

（如果你有想参考的其他公众号，记录在这里。）
```

## Phase 3g: 更新 state

```json
{
  "historical_samples": {N},
  "calibration_samples": 0,
  "cold_start_remaining": 5,
  "mode": "cold-start",
  "rubric_version": "v0 | v1",
  "baseline_metrics": {
    "avg_reads": {N},
    "share_rate": {X.X},
    "collection_rate": {X.X},
    "comment_rate": {X.X},
    "like_rate": {X.X},
    "calculated_at": "ISO",
    "sample_count": {N}
  },
  "rubric_fitted_from_history": true | false
}
```

### Mode 判定

```
if calibration_samples >= 5:
    mode = "calibration"
elif historical_samples >= 5 AND rubric_fitted_from_history:
    mode = "cold-start"
else:
    mode = "cold-start"
```

> **关键**：即使有 100 篇历史文章，只要 calibration_samples = 0，mode 就是 cold-start。预测格式是简化版（不打 bucket 数字），但 rubric 可以已经是 v1 加权。

更新 state 后，**必须向用户解释此区别**（见 grindraft-init Phase 6 输出模板），避免用户看到"cold-start"后困惑"我明明导入了 8 篇文章"。

## Phase 3h: 自检

写完所有数据后，逐项重读验证：

| # | 目标 | 读什么 | 通过条件 |
|---|------|--------|---------|
| 1 | 文章目录 | `articles/` | 每篇历史文章都有 `final.md` + `prediction.md` + `retro/report.md` |
| 2 | style_guide.md | `style_guide.md` | 含"历史文章分析"段，风格参数已更新 |
| 3 | benchmark.md | `benchmark.md` | 含基准指标表，数据非空 |
| 4 | rubric_notes.md | `rubric_notes.md` | 如拟合成功，含 v1 公式；如失败，含偏差记录 |
| 5 | state.json | `.grindraft-state.json` | `historical_samples` = N，`baseline_metrics` 非空 |

### 自检输出格式

```
📋 历史文章导入自检 Phase 3h:
  □ articles/ → {N} 篇文章已落盘 ✅
  □ style_guide.md → 历史文章分析段已写入，{M} 条 pattern ✅
  □ benchmark.md → 基准指标已设定 ✅
  □ rubric_notes.md → rubric {v0/v1}，拟合{'成功'/'跳过'} ✅
  □ state.json → historical_samples={N}，baseline_metrics 已写入 ✅

✅ 历史文章导入自检全部通过
```

## 与后续流程的关系

### 对 grindraft-write 的影响

- 写稿时读 style_guide.md → 自动应用从历史文章提取的风格规则
- 历史分析出的"绝对禁区"和"推荐口语化词组"立即生效

### 对 grindraft-predict 的影响

- mode 仍为 cold-start → 前 5 篇新文章用简化预测
- 但 rubric 可能是 v1（如果拟合成功）→ 7 维打分用加权而非等权
- 锚点对比可以用历史文章（标 retrospective）作为参照

### 对 grindraft-retro 的影响

- 复盘时"vs 基准"有数据了——不再是"基准值用粉丝量相近账号的均值"
- 派生比率（分享率/收藏率等）可以直接和用户自己的历史基准对比

### 对 grindraft-bump 的影响

- 如果拟合出了 v1，后续 bump 从 v1 出发而非 v0
- 拟合的权重 + 后续 blind prediction 复盘数据 → bump 时样本更丰富
