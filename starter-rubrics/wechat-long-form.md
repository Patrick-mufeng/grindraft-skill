# 公众号长文 Rubric — 7 维评分体系

> **适用内容形态**：公众号长文（1500-10000 字）。
> **版本**：v1（已校准，基于 5 篇复盘数据）。权重从数据中拟合。
> **详细维度定义**：见 [shared-references/rubric-dimensions.md](../shared-references/rubric-dimensions.md)。

## 维度与权重

| 维度 | 简称 | 权重 | 测什么 |
|---|---|---|---|
| 标题钩力 | HK | ×0.5 | 触达瓶颈下已不驱动阅读量 |
| 叙事牵引力 | NR | ×1.0 | 完读率的基底（稳定） |
| 金句锚点 | QA | ×1.0 | 社交传播的内容基础 |
| 实用密度 | UT | ×2.5 | 完读率+收藏的最强驱动力（✅ 5/5） |
| 思辨激发度 | DT | ×0.5 | 触达瓶颈下留言始终为 0 |
| 情绪峰值 | EP | ×2.0 | 分享率的最强驱动力（✅ 首次验证 23.5%） |
| 结构闭环 | SC | ×0.5 | 从"好文章"到"作品"的加分项 |

## Composite 公式

```
raw = (HK×0.5 + NR×1.0 + QA×1.0 + UT×2.5 + DT×0.5 + EP×2.0 + SC×0.5) / 8.0
composite = raw × 2.0
```

composite 范围：0-10（每个维度 1-5 × 权重归一化 × 2.0）。与 cheat-on-content 量纲一致。

## Bucket 映射

| composite 区间 | 预测 bucket |
|---|---|
| ≥ 8.4 | S（>50,000 阅读） |
| 6.8-8.3 | A（10,000-50,000） |
| 4.8-6.7 | B（3,000-10,000） |
| 3.2-4.7 | C（1,000-3,000） |
| 1.6-3.1 | D（300-1,000） |
| < 1.6 | E（<300） |

> Bucket 边界随用户粉丝基数调整。grindraft-init 时按 `follower_count` 偏移：follower_count 每 +1000，bucket 下界 +10%。

## 打分流程

1. 通读全文一遍（不停、不分析——感受"作为一个读者的完整体验"）
2. 第二遍逐维打分（每维 1-5，一行理由引用原文具体句子/结构）
3. 算 composite
4. 按 composite 推 bucket + 中枢
5. 找锚点对比（历史预测中 composite 接近的样本）
6. 写反事实场景 + 关键校准假设

## 注意事项

- **标题钩力（HK）打分要读标题本身**——不是文章内容。标题不好但内容好 ≠ HK 高。
- **叙事牵引力（NR）关注"节奏"**——读者在哪段可能弃文？段落太长？转折生硬？
- **金句锚点（QA）关注"可摘抄性"**——不是所有好句子都是金句，金句要独立成味、被截屏分享后仍有冲击力。
- **实用密度（UT）关注"可操作性"**——不是信息量大就叫实用。读者能直接拿去用的才算。
- **思辨激发度（DT）关注"争议空间"**——完全正确的文章没有留言冲动。适度留白、抛开放问题。
- **情绪峰值（EP）只认真正的峰值**——普通的共情不算，要找那个"停下来，需要消化一下"的瞬间。
- **结构闭环（SC）看契诃夫之枪**——开头提到的意象/问题/人物，结尾有没有回来？
