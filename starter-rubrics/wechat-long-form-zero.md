# 公众号长文 Rubric — v0 等权版（Cold-start）

> **适用场景**：用户前 5 篇预测。不做加权 composite，只给等权平均 + 方向押注。
> **设计意图**：没校准数据时加权是 false precision——等权更诚实。

## 维度与权重

所有 7 维等权（×1.0）：

| 维度 | 简称 | 权重 |
|---|---|---|
| 标题钩力 | HK | ×1.0 |
| 叙事牵引力 | NR | ×1.0 |
| 金句锚点 | QA | ×1.0 |
| 实用密度 | UT | ×1.0 |
| 思辨激发度 | DT | ×1.0 |
| 情绪峰值 | EP | ×1.0 |
| 结构闭环 | SC | ×1.0 |

## Composite 公式（v0）

```
raw = (HK + NR + QA + UT + DT + EP + SC) / 7
composite = raw × 2.0
```

composite 范围：0-10（每个维度 1-5 × 等权平均 × 2.0）。与 cheat-on-content v0 量纲一致。

## Cold-start 简化预测

前 5 篇不打 bucket 数字、不给概率分布、不给反事实场景。只写：

1. **7 维等权打分**（每维 1-5 + 一行理由）
2. **方向押注**：比上一篇"明显好 / 差不多 / 可能不如"
3. **锚点 N/A 说明**：显式标"这是第 N 篇预测，尚无同类型锚点"

第 5 篇复盘后 → grindraft-status 提示"完整预测模式已解锁"，切换到 `wechat-long-form.md`（v1 加权版）。

## 为什么等权？

- 初始权重是先验猜测——可能完全错
- 等权让 cold-start 期的预测误差更透明
- 5 篇复盘后，权重的调整方向有数据支撑，而不是"我一上来就觉得 HK 该 ×2"

## 从 v0 升级到 v1 的触发条件

- calibration_samples ≥ 5
- 且至少 3 篇的 composite 排序与实际阅读量排序一致（证明 rubric 有基本预测力）

不满足 → 继续 cold-start，同时提示用户在 rubric_notes.md 里记录偏差方向。
