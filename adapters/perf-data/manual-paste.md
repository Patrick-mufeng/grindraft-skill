# Manual Paste Adapter

> 复盘数据手动粘贴适配器。grindraft-retro 的默认数据收集方式。
> 公众号后台数据目前无公开 API——这是唯一可靠的数据源。

## 数据收集流程

grindraft-retro Phase 1 触发：

1. 提示用户打开公众号后台 → 内容管理 → 已发布 → 找到对应文章
2. 要求用户粘贴以下数据：

```
阅读量：_______
分享数：_______
在看数：_______
留言数：_______
收藏数：_______
阅读完成率：______%
```

3. 要求用户从留言区复制 Top 20 高赞留言（每条带赞数）

4. 将原始数据写入 `retro/<date>_<id>_<short>/report.md`

## 数据解析规则

- 数字格式自动识别：支持"1.2万"/"12,000"/"12000" 等格式
- 如果某字段缺失 → 标 `N/A`，不阻塞流程
- 必填字段：阅读量（核心校准信号）——缺失则拒绝继续

```markdown
# 复盘数据报告 — <文章标题>

**数据来源**：公众号后台手动粘贴

## 派生比率

- 分享率：分享 / 阅读 = X%
- 在看率：在看 / 阅读 = X%
- 留言率：留言 / 阅读 = X%

## Top 20 留言

（每条带赞数，按赞数降序）
```

## 失败模式

| 错误 | 处理 |
|---|---|
| 用户拒绝粘贴数据 | **拒绝继续**——没有数据无法复盘 |
| 部分字段缺失 | 标 N/A，继续 |
| 用户只给阅读量 | 提醒"评论和分享是更重要的信号"，但仍接受 |
| 用户给 <5 条留言 | 警告"评论是判断 DT/EP 的关键信号"，但不阻塞 |

## 稳定性等级

★★★★★（不依赖外部 API，永远可用）
