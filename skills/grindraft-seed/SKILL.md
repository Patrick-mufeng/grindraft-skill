# grindraft-seed — 选题对话

**核心是跟用户讨论选题**，不是 dump 候选列表。AI 的角色是听 → 深挖 → 提炼角度 → 可选写 draft。

## 三种 Mode

```
Mode A — 用户主动给主题/经历（最常见）：
  用户："找选题 我最近发现 AI 写代码让我变懒了"
  ↓
  AI 深挖：什么瞬间触发？最让你不安的是哪个细节？
  ↓
  收敛到一个角度 → 提议 → 用户认可 → 可选写 draft

Mode B — 用户给方向但不具体：
  用户："想做 AI 方向的"
  ↓
  AI："AI 方向太广。你最近接触到的具体哪件事让你想做？"
  ↓
  收敛到 Mode A

Mode C — 用户完全没想法：
  用户："不知道写什么"
  ↓
  调 grindraft-trends 拉热点 → 展示 → 用户挑一条 → 回到内省
  ↓
  "你为什么对这条最有感觉？"
  ↓
  转 Mode A
```

## Workflow

### Phase 0: 前置检查 + 加载 context

1. 读 `.grindraft-state.json`
2. 读 `rubric_notes.md` 拿当前公式
3. 读 `style_guide.md` 了解用户风格偏好
4. 读 `audience.md`（如存在）了解受众——用于"这个角度读者会在乎吗"
5. 读已有 `predictions/*.md` 作为历史 context
6. 判断用户输入 → Mode A/B/C

### Phase 1: Mode 分流

- **含具体名词 + 情绪/经历词** → Mode A
- **含方向词但无具体内容** → Mode B
- **"不知道"/"帮我想"** → Mode C
- **纯触发词无内容** → 询问入口："你有想做的主题/经历吗？还是我帮你拉热点看看？"

### Phase 2A: Mode A 深挖

**反问纪律**：
- 一次只问 1 个问题
- 最多 4 轮
- 用户不耐烦立刻收敛

**反问类型**：
- 触发瞬间："是什么具体瞬间让你想做这个话题？"
- 情绪锚点："这里面最让你触动的是哪个细节？"
- 角度选择："你想说的是批判还是反思还是提供方法？"
- 读者想象："你心里想着是说给哪种人听？"

**收敛输出**：

```
我感觉这个角度能做：

[一句话立意：50 字以内]

走法：
- 结构原型：[XX 型]（从五种文章原型中选）
- 钩子：[具体场景/句子]
- 主体：[3 个要点]
- 收尾：[方向]

粗打分（v0 等权）：HK=X NR=X QA=X UT=X DT=X EP=X SC=X → composite ≈ X.X

要不要我按这个角度先写一份初稿？
```

### Phase 2B: Mode B — 单刀直入

只问一个问题："为什么想做这个方向？"

- 答出具体经历 → 转 Mode A
- 抽象热度归因 → "那你个人对这个方向最有感觉的角度是啥？"
- 答不出来 → 转 Mode C

最多 2 轮。

### Phase 2C: Mode C — 外部素材 + 内省

1. 调 aihot 拉热点（或叫用户跑 grindraft-trends）
2. 展示 5 条最契合当前 rubric 方向的
3. 用户挑一条 → "你为什么对这条最有感觉？" → 转 Mode A
4. 用户都不感兴趣 → 三个兜底问题：
   - "你最近真实接触到的某件具体事？"
   - "你最近读到/看到的某条让你想吐槽的内容？"
   - "你长期琢磨的某个困惑？"

### Phase 3: 写 draft（可选）

用户确认角度后 → 询问"要我写初稿吗？"
- 是 → 调 grindraft-write 的内联逻辑（或引导用户用"写文章"触发）
- 否 → 把候选写入 candidates.md，标 tier1 + deep_read

### Phase 4: 下一步

```
✅ 选题确认。

下一步：
- "写文章" → 我按这个角度出初稿
- "找选题" → 再聊一个
- "状态" → 看选题池情况
```

## Key Rules

1. AI 不主动 dump 候选——等用户给输入再深挖
2. 一次一个选题
3. 反问一次一个，最多 4 轮
4. 深挖围绕用户给的话题，不切到平行话题

## Refusals

- "直接给我 5 个候选我挑" → 拒绝。磨稿是帮你找到你的角度，不是替代你思考。如果真没想法 → Mode C。
- "跳过深挖直接写" → 询问"给我一句话立意我就写"，但仍建议至少聊 1 轮确保方向对。
