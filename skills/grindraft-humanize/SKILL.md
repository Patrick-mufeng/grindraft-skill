# grindraft-humanize — 四层去 AI 味

> 方法论继承自 khazix-writer 的 L1-L4 自检体系，**去掉卡兹克个人风格的绑定**，做成通用版。
> 默认先跑通用规则。用户的个人禁区从 `style_guide.md` 读取并叠加。

## Overview

```
[用户：去 AI 味 articles/{标题}_{日期}/draft.md]
  ↓
[Phase 0: 读 style_guide.md 拿个人禁区]
  ↓
[Phase 0.5: 风格沉淀 — diff 当前终稿 vs AI 初稿，去重写入 style_guide]
  ↓
[Phase 1: L1 硬规则扫描]  ← 沉淀出的个人禁区已在此生效
  ↓
[Phase 2: L2 风格一致检查]
  ↓
[Phase 3: L3 内容质量审查]
  ↓
[Phase 4: L4 活人感终审]
  ↓
[Phase 5: 输出质检报告 + 备份原稿 + 覆盖文件]
  ↓
[Phase 6: 自检 — 逐项重读验证全部写入]
```

## Phase 0: 读用户个人禁区

1. 读 `style_guide.md` 的"绝对禁区"段——这是用户在使用中积累的个人雷区
2. 将个人禁区**追加**到 L1 的扫描列表（通用规则 + 个人规则）
3. 读"推荐口语化词组"段——在 L2 检查中使用

如果 style_guide.md 的禁区段为空 → 提示"你还没有个人禁区列表。每次改稿时告诉我你特别不喜欢的表达，我会记到 style_guide.md 里。"

## Phase 0.5: 风格沉淀（改稿 pattern 自动提取）

在去 AI 味之前，先 diff 用户当前版本 vs AI 初稿，提取用户的改写 pattern 写入 style_guide.md。
这样 L1 扫描就能立刻用上刚沉淀的个人禁区。

按 [shared-references/style-precipitation.md](../../shared-references/style-precipitation.md) 执行：

1. **检查是否需要沉淀**：读 `.grindraft-cache/precipitation-log.jsonl`，看本文最近一次沉淀时的 file_hash
2. **计算当前 hash**：`articles/{标题}_{日期}/final.md` 或 `draft.md` 的正文 sha256[:12]
   - hash 一致 → 跳过（无新改动）
3. **Diff**：当前版本 vs 上一轮沉淀时的快照（首轮 vs AI 初稿 draft.md）
4. **识别 pattern**（砍掉的 AI 词/新增的口语表达/结构偏好）
5. **去重写入 style_guide.md**：逐条比对，已有则跳过，同义变体标"待验证"
6. **记录日志**：`{"article": "{标题}_{日期}", "round": N, "file_hash": "...", "patterns_written": [...], "precipitated_at": "ISO"}`
7. **更新快照**：将当前文件复制为 `plates/style-diffs/{标题}_{日期}_precipitated.md`（下次 diff 的基准）

> **纪律**：只沉淀 ≥2 次出现的 pattern。单次改动标"待验证"不写入正式规则。

这一层是绝对不会被接受的东西。**任何一项不通过就必须修复，没有例外。**

### L1-1 通用禁用词扫描

全文搜索以下词汇，出现则必须替换：

| 禁用 | 替换方向 |
|---|---|
| "首先...其次...最后" | 用自然转场替代 |
| "综上所述" / "总的来说" | 删掉，或用具体回扣句 |
| "值得注意的是" / "不难发现" | 删掉，直接说 |
| "在当今...的时代" / "随着...的发展" | 改成具体场景切入 |
| "让我们来看看..." / "接下来让我们..." | 删掉，直接说 |
| "本质上" / "换言之" / "换句话说" | 换成更口语的表达 |
| "不可否认" / "众所周知" | 删掉，直接陈述 |

### L1-2 结构性套话扫描

- 连续 bullet point ≥4 条且每条 ≥30 字 → 改成散文叙述或缩短
- 大段加粗（>2 行连续加粗）→ 只保留核心结论加粗
- 结尾喊口号（"让我们一起..."）→ 改成具体建议或留白

### L1-3 用户个人禁区扫描

从 `style_guide.md` 读取用户个人禁区，逐条扫描。

### L1-4 格式问题修复

- 中英文之间自动加空格（如果需要）
- 标点：将英文逗号在中文上下文中替换为中文逗号
- 破折号：中文上下文中 "—" → 根据用户偏好处理

**通过标准**：零命中。
**修复方式**：逐个替换，不整体重写。

## Phase 2: L2 风格一致检查

逐项"是/否"判断。每项至少 3/4 通过。

### L2-1 开头检查
- [ ] 是否从一个具体的场景/事件/行为切入？
- [ ] 第一句是否让人产生"然后呢？"的冲动？
- [ ] 是否避免了一切教科书式/宏大叙事开头？

### L2-2 节奏与结构
- [ ] 是否有长短句交替？连续 3 句长度相近 = 节奏呆板
- [ ] 是否有一句话独立成段的"断裂"效果？（全文至少 2-3 处）
- [ ] 偏离主线的段落后是否有"扣主线句"拉回来？
- [ ] 板块之间是否有口语化转场（而非 markdown 小标题堆叠）？

### L2-3 口语化检查
- [ ] 是否使用了口语化过渡词？（至少 5 个不同的——"说实话""你想想看""问题来了""说白了就是"等）
- [ ] 是否有至少一处自嘲或承认不确定？（"说实话我也不确定""我自己也还在想"）
- [ ] 是否有使用疑问句制造节奏刹车？

### L2-4 结尾检查
- [ ] 是否避免了"让我们共同期待""相信未来会更好"等口号式结尾？
- [ ] 是否要么回扣开头、要么给出具体行动建议、要么留白？

**通过标准**：L2-1 全部通过，L2-2 至少 3/4，L2-3 至少 3/4，L2-4 全部通过。

## Phase 3: L3 内容质量审查

### L3-1 观点支撑
- [ ] 每个核心观点是否都有具体细节/场景/案例支撑？
- [ ] 有没有空泛的论断只有结论没有例证？

### L3-2 知识输出方式
- [ ] 知识点是以"聊天式掏出"还是"教科式介绍"？
- [ ] 如果有引用（论文/数据/他人观点），是否自然融入而非"插播"？

### L3-3 对立面与同理心
- [ ] 在讲核心观点时，是否有对对方立场的理解？
- [ ] 是否先站到读者的处境里，再给出自己的视角？

### L3-4 文章类型专项
按文章原型做针对性检查：
- 解读型 → 是否有"观察→好奇→研究→升维"的推进？
- 工具型 → 是否有个人故事铺垫？效果展示是否具体？
- 体验型 → 是否有"亲自下场"的叙事感？
- 方法论型 → 每节是否落到可执行行动？是否说明了学习成本？
- 观点型 → 立场是否鲜明？是否有证据支撑？

**通过标准**：L3-1 和 L3-2 全部通过，L3-3 和 L3-4 至少通过适用项。

## Phase 4: L4 活人感终审

以读者的视角通读全文，回答一个核心问题：

**"读完这篇文章，我感觉是一个有见识的人在认真跟我聊一件他关心的事，还是一个 AI 在给我输出信息？"**

### L4-1 温度感
- 情绪表达是体感记忆（"我当时就愣住了"）还是知识描述（"我感到非常震撼"）？
- 如果有提到具体的人/场景，这个描写是否让人"能感觉到温度"？

### L4-2 独特性
- 这篇文章是否有"只有这个人才能写出来的角度"？
- 还是换一个人+换一个 AI 也能写出差不多的？

### L4-3 姿态
- 文章的语气是不是"一个有见识的普通人在聊一件打动他的事"？
- 有没有滑入"导师在教学生"或"品牌在做营销"的姿态？

### L4-4 心流
- 从头到尾读，有没有哪个地方注意力断掉了？需要回头理解逻辑？
- 如果有——那个地方就是需要修复的节奏或逻辑问题。

**通过标准**：L4-1 到 L4-4 整体感觉"这像是真人写的"。任何一项让你觉得"这段 AI 味太重了" → 返回修复。

（风格沉淀已在 Phase 0.5 完成 — 见上方 Phase 0.5）

```

- 结构套话：N 处（已修复）
- 个人禁区：N 处（已修复）

**L2 风格一致** ✅ / ❌
- 开头：✅
- 节奏：✅（具体问题：...）
- 口语化：✅（使用了 N 个不同口语词组）
- 结尾：✅

**L3 内容质量** ✅ / ❌
- 观点支撑：✅
- 知识输出：✅
- 对立面与同理心：✅
- 类型专项：✅

**L4 活人感** ✅ / ❌
- 温度感：✅（具体表现：...）
- 独特性：✅
- 姿态：✅
- 心流：✅（或"断点在第 N 段，已修复"）

**总评**：四层全部通过 / X 层需要返工
```

**落盘**：

1. **备份原稿**：覆盖 draft 前，先把去味前版本复制到 `plates/style-diffs/{标题}_{日期}_before.md`。这确保后续 diff 和 style_guide 沉淀有源数据。
2. 覆盖原 draft 文件为 `articles/{标题}_{日期}/final.md`（去味后即为终稿）。

### Phase 6: 自检 — 逐项重读验证写入完整性

写完所有数据后，**重新读取每个目标文件**，逐项确认写入生效。

#### 验证清单

| # | 目标 | 读什么 | 通过条件 |
|---|---|---|---|
| 1 | draft 文件 | `articles/{标题}_{日期}/final.md` | 文件存在，L1-L4 修复已体现在正文中 |
| 2 | 备份文件 | `plates/style-diffs/{标题}_{日期}_precipitated.md` 或 `{标题}_{日期}_before.md` | 文件存在，内容与当前版一致 |
| 3 | style_guide.md（如有写入） | `style_guide.md` | 如 Phase 0.5 有新 pattern，已在文件中可找到 |
| 4 | style_guide 改稿历史 | `style_guide.md` 的"改稿历史观察"表 | 如本次做了额外手动修改，已追加新行 |
| 5 | precipitation 日志（如有写入） | `.grindraft-cache/precipitation-log.jsonl` | 如 Phase 0.5 有沉淀，日志行存在 |

#### 自检输出格式

```
📋 自检 Phase 6:
  □ articles/{标题}_{日期}/final.md → L1-L4 修复已落盘 ✅
  □ plates/style-diffs/ → 沉淀快照已保存 ✅
  □ style_guide.md → 无新 pattern 写入，跳过
  □ style_guide.md → 改稿历史已追加 ✅
  □ precipitation-log.jsonl → 本轮沉淀已记录 ✅（或"无新沉淀，跳过"）

✅ 去 AI 味自检全部通过
```

#### 失败处理

- ❌ draft 未更新 → **立即重写**
- ❌ 备份未写入 → **立即补写**
- ❌ style_guide 未同步 → **立即补写**
- 重写后再次 re-read 确认 → 仍不通过 → 输出 `❌ 自检失败，请手动检查`

## Key Rules

1. **通用规则优先**——先跑 L1-L4 通用版，再叠用户个人禁区
2. **用户个人禁区从 style_guide.md 读**——不硬编码任何人的偏好
3. **L4 不能用机械方法**——活人感是"读"出来的，不是"扫"出来的
4. **修完一层再过下一层**——L1 没过的别跳到 L2

## Refusals

- 「跳过 L4，我只想扫错别字」 → 拒绝。L4 是去 AI 味的核心——"活人感"不是错别字能覆盖的。
- 「把 AI 语气全删了，但别动我的内容」 → 这就是四层自检在做的事——放心。
- 「你的 L2 检查把我的风格改没了」 → L2 只检查"是否有口语化特征"，不强制要求特定的风格。如果你觉得某处被改坏了 → 告诉我你更喜欢原来的表达，我会记到 style_guide.md。
