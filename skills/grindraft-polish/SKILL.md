# grindraft-polish — 标题 + 简介 + 封面提示词

文章写完了，但"好文章"和"好内容"之间差了三样东西：一个让人点进来的标题，一段让人想转发的简介，一张在信息流里让人停下来的封面。

这个 skill 做这三件事。

## Overview

```
[用户：打磨标题 articles/{标题}_{日期}/draft.md]
  ↓
[Phase 0: 读文章 + 理解核心]
  ↓
[Phase 1: 生成 4-5 个标题候选]
  ↓
[Phase 2: 生成简介（一句话 + 一段描述）]
  ↓
[Phase 3: 生成 2-3 套封面图提示词]
  ↓
[Phase 4: 用户挑选 + 写入文件 header]
```

## 标题钩子公式（参考 baoyu-format-markdown 的 title-formulas）

| # | 公式 | 特征 | 适合 |
|---|---|---|---|
| 1 | **颠覆式** | 否定常见认知，制造认知冲突 | 挑战主流观点的文章 |
| 2 | **解决方案** | 直接给答案，承诺具体价值 | 教程、干货、方法论 |
| 3 | **悬念** | 只说一半，制造好奇心缺口 | 个人故事、案例拆解 |
| 4 | **具体数字** | 用数字增加可信度和冲击力 | 数据驱动、评测对比 |
| 5 | **对比** | 小原因→大结果，或预期 vs 现实 | 发现类、对比类、反转类 |
| 6 | **结果先行** | 先抛惊人结果，钩读者找原因 | "我试了 X"型文章 |
| 7 | **反问** | 用问题制造未完成感 | 问题意识型、诊断型内容 |
| 8 | **共情** | 触碰痛点，触发"对对对我也这样" | 经历分享、困境叙事 |
| 9 | **直白** | 清晰陈述主题，不绕弯 | 所有类型，作备选 |

## Workflow

### Phase 0: 读文章 + 提取关键信息

1. 读文章全文
2. 提取：
   - **核心论点**：一句话，这篇文章到底在说什么
   - **最冲击力的数据/事实**：如果有具体数字、反常识发现
   - **情绪基调**：愤怒？惊奇？担忧？兴奋？共情？
   - **目标读者**：谁会点进来？开发者和创业者想要的是完全不同的标题

### Phase 1: 生成 4-5 个标题候选

**生成规则**：

1. 从 9 种公式中选 4-5 个最适合文章调性的，各出一个标题
2. 每个标题标注用了哪个公式
3. 标注推荐理由（一句话）
4. 标记最推荐的 1 个（⭐ ）

**标题原则**：
- 前 5 个字制造信息缺口或认知冲突
- 具体 > 抽象（"594%" > "大幅提升"）
- 否定 > 肯定（"你不是 XX" > "你应该 XX"）
- 对话感，不像论文标题
- 控制在 30 字内（公众号标题过长会截断）
- 准确不标题党——文章得接得住标题的承诺

**展示格式**：

```
📝 标题候选

1. [颠覆式] 「你看到 Codex 暴涨 594%，我看到英伟达在锁你的未来」
   → 双重视角+认知冲突，适合深度读者

2. [结果先行] 「594%——英伟达刚放了个大招，但你可能没看懂」
   → 数字+悬念，适合大众读者

3. [对比] 「同一个 4B 模型，用了 Polar 之后 Codex 从废物变成学霸」
   → 反差对比，有画面感

4. [直白] 「英伟达 Polar 框架：让所有编程智能体原地提升 594%」
   → 信息明确，适合 SEO 和严肃读者

5. [共情] 「如果你也在追 AI 工具更新——这篇会让你停下来想三秒」
   → 触碰"信息焦虑"痛点

⭐ 推荐 #1 或 #2——取决于你想让谁点进来
```

### Phase 2: 生成简介

生成两份简介，用于不同场景：

**一句话简介（50-80 字）**：
- 公众号分享时的摘要
- 要求：一句话说清楚价值，不剧透最好部分
- 公式：核心事件 + 意外发现 + 为什么值得看

**一段描述（100-200 字）**：
- 朋友圈转发文案或文章导语
- 允许适当悬念和情绪
- 公式：开头钩子 + 核心内容 + 发人深省的问题/观点

```
📋 文章简介

一句话：英伟达新框架 Polar 让 Codex 跑分暴涨 594%——但真正的故事不是数字，是谁在控制 AI 淘金热里的铲子。

一段描述：
今天 AI 圈被一个数字刷屏：594.74%。英伟达 Polar 框架让 Codex SWE-Bench 从 3.8% 跳到 26.4%。但读完论文我发现——Polar 不只是一个"让模型变聪明"的工具。它是英伟达在 AI 淘金热里的新铲子。而且这把铲子，可能比 GPU 更值钱。
```

### Phase 3: 生成封面图提示词

**不生成图片**——只输出提示词，用户可喂给 Midjourney / DALL·E / SD / 微信后台 AI 封面生成。

**生成 2-3 套**，每套包含：

| 维度 | 说明 |
|---|---|
| **类型** | 抽象概念 / 具象场景 / 文字主导 / 极简图形 |
| **色板** | 主色 + 辅色 + 情绪，参考封面规范 11 套色板但自由发挥 |
| **构图** | 主体位置、留白、视觉焦点 |
| **情绪** | 科技感 / 紧迫感 / 神秘感 / 温暖感... |
| **文字建议** | 如果封面要放文字，放什么、放哪 |

**提示词语言**：中英双语。英文 prompt 给生图工具，中文描述给用户理解。

```
🎨 封面提示词

方案 A — 抽象概念型
  中文：深色背景上一个巨型金色铲子插入电路板地面，铲子周围爆出绿色数据光点。
       远处是一条金色矿脉延伸向地平线。科技感+淘金隐喻。
  英文：A giant golden shovel piercing into a circuit board ground, green data particles bursting around the shovel, a golden ore vein stretching to the horizon in the distance. Dark navy background. Tech illustration style, 3D render, dramatic lighting, 16:9 aspect ratio, minimalist composition.

  色板：深海军蓝 #0a0e1a + 金属金 #d4a84b + 荧光绿 #00e676
  构图：铲子居中偏右，矿脉从左下到右上方延伸
  情绪：科技硬核 + 隐喻感
  字：无字版本；如需加字，左下角放 "铲子" 或 "SHOVEL"

方案 B — 文字主导型
  中文：纯黑背景。巨大的 "594%" 数字以电光青色渲染，数字间隙中嵌入细小的代码片段。
       右下角一行极小的 "NVIDIA Polar"。
  英文：Pure black background. Giant "594%" in electric cyan neon typography, tiny code snippets embedded between the digits. "NVIDIA Polar" in small white text at bottom right. Cyberpunk minimalism, typography-first design, 16:9.

  色板：纯黑 #000 + 电光青 #00e5ff + 白 #fff
  构图：594% 占画面 70%，居中偏上，底部留呼吸空间
  情绪：冲击力、极简、数据驱动
  字：本身就是文字封面

方案 C — 具象场景型
  中文：一个矿工站在巨大的 GPU 芯片峡谷边缘，峡谷底部是代码河流在发光。
       矿工背对镜头，手里拿的不是镐，是一根光缆。
  英文：A miner standing at the edge of a giant GPU chip canyon, a glowing river of code flowing at the bottom. The miner faces away, holding not a pickaxe but a fiber optic cable. Dramatic scale, cinematic lighting, dark teal and amber color palette, 16:9.

  色板：深青 #0d1f1a + 琥珀金 #d4943a + 暗蓝灰 #1a2a3a
  构图：人物在左下 1/3 处，芯片峡谷占右和上方
  情绪：史诗感、隐喻、探索
  字：无字
```

### Phase 4: 用户挑选 + 落盘

用户选择后：

1. **标题**：写入文章 frontmatter 的 `title` 字段（或更新已有）
2. **简介**：写入 `summary` 和 `description` 字段
3. **封面提示词**：写入 `cover_prompts/` 目录下的独立 `.md` 文件
4. 在文章 header 中追加元信息

如果用户不选——全部保留到 `articles/{标题}_{日期}/polish.md` 供后续参考。

## Key Rules

1. **标题不标题党**——文章必须接得住标题的承诺。数字要准确，观点要文章里有
2. **提示词不生成图**——只出 prompt，不调任何图片 API。用户自己拿到 Midjourney/DALL·E/微信后台去跑
3. **每套提示词独立完整**——不用依赖其他方案的上下文就能直接拿去用
4. **色板写 hex 值**——`#0a0e1a` 不是"深蓝色"，方便用户直接取色做统一视觉
5. **语言**：提示词中英双语，中文给用户看，英文给生图工具

## Integration

- 上游：`grindraft-write` 或 `grindraft-humanize` 产出终稿
- 下游：`grindraft-format` 排版时使用选定的标题和简介
- 侧出：封面提示词写入独立文件，不进入排版流程
