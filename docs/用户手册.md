## 1. 这是什么

**磨稿不是"AI 代写工具"**。它是帮你建立一套**越写越准的写作系统**。

普通的 AI 写作：你说"写一篇关于 AI 的文章"→ AI 写了一篇 → 完了。
下一次：你又说"写一篇关于 AI 的文章"→ AI 又写了一篇 → 和上次毫无关系。

磨稿的做法：写 → 预测阅读量 → 发布 → 复盘真实数据 → 进化评分系统 → 下次预测更准。

```
第 1 篇：预测 3000 阅读，实际 5200——低估了，为什么？
第 2 篇：预测 4500 阅读，实际 4100——接近了
第 3 篇：预测 3800 阅读，实际 3900——越来越准
...
第 20 篇：预测误差在 ±20% 以内
```

**每篇都让系统更了解你的读者、你的风格、你的"爆款公式"。**

## 2. 快速上手（5 分钟）

### 第一步：安装

把 `grindraft/` 目录放到你的 Reasonix skills 目录：

```
# 项目级（只在这个项目可用）
cp -r grindraft/ <你的项目>/.reasonix/skills/

# 或全局（所有项目可用）
cp -r grindraft/ ~/.reasonix/skills/
```

### 第二步：初始化

在你的文章项目目录下，打开 Reasonix Code，说：

```
磨稿初始化
```

跟着 AI 回答几个问题：

```
AI: 你公众号有多少关注？
你: 4000

AI: 你的写作状态是？
    A) 有粉丝但文章不是自己写的，想沉淀个人风格
    B) 从零开始
    C) 已经在写，想用磨稿校准
你: A

AI: 发布频率？
你: 周 2 篇

AI: AI 参与程度？
    A) low——我写核心，AI 只润色
    B) medium——我给大纲，AI 扩写
    C) high——我给选题，AI 出初稿
你: C
```

初始化完成。你的项目目录下会多出一堆文件——别慌，往下看。

### 第三步：写第一篇文章

```
找选题                      # 聊聊你想写什么
写文章                      # AI 出初稿
去 AI 味                    # 四层质检
设计封面                    # 40 套模板选一个生成封面
排版                        # 转公众号 HTML
启动预测                    # AI 打分，预测阅读量
（你复制 HTML 到公众号后台，发布）
已发布                      # 登记一下
（3 天后）
复盘                        # 粘数据，看预测准不准
```

**恭喜，闭环跑通了。** 以上每一步的详细说明见第 4 章。

## 3. 架构总览

### 3.1 三层结构

```
┌──────────────────────────────────────────────────────┐
│  SKILL.md — 总协议 + 路由表                            │
│  "用户说 X → 调 grindraft-Y"                          │
├──────────────────────────────────────────────────────┤
│  14 个子 Skill（每个是一个独立的 SKILL.md）             │
│  init / trends / seed / write / humanize / format     │
│  predict / score-blind / publish / retro /            │
│  persona / bump / recommend / status                  │
├──────────────────────────────────────────────────────┤
│  基础层：共享协议 + 模板 + 评分规则 + 适配器            │
│  shared-references/ (7) + templates/ (7)              │
│  starter-rubrics/ (2) + adapters/ (2)                 │
└──────────────────────────────────────────────────────┘
```

### 3.2 子 Skill 地图

```
入口
  │
  ├─ grindraft-init         【初始化】创建项目骨架、配置参数
  │
  ├─ grindraft-status       【看板】任何时候可调，看整体状态
  │
  ▼ 写作闭环
  ├─ grindraft-trends       【找热点】从 aihot 拉 AI 圈资讯
  ├─ grindraft-seed         【找选题】对话深挖，收敛角度
  ├─ grindraft-write        【写初稿】AI 出稿（3 种参与度）
  ├─ grindraft-humanize     【去 AI 味】四层质检 (L1→L4)
  ├─ grindraft-illustrate   【配图】小霓赛博风格正文插图，API 生图
  ├─ grindraft-cover        【封面】40 套模板自动匹配生成封面
  ├─ grindraft-format       【排版】MD → 公众号 HTML
  │
  ▼ 校准闭环
  ├─ grindraft-predict      【盲预测】7 维分 + bucket + 概率
  ├─ grindraft-score-blind  【隔离打分】内部 sub-agent，防数据污染
  ├─ grindraft-publish      【发布登记】记 URL 和时间
  ├─ grindraft-retro        【复盘】收数据 + 对比预测 + 提炼观察
  │
  ▼ 进化
  ├─ grindraft-persona      【受众画像】从留言派生"谁在看"
  ├─ grindraft-bump         【升级公式】调整权重/bucket/维度
  └─ grindraft-recommend    【推荐选题】按 composite 排序
```

### 3.3 数据流向

```
用户输入（选题/经历/大纲）
     │
     ▼
grindraft-write ────→ articles/{id}/draft.md（AI 初稿）
     │
     ▼（用户改写 + 去 AI 味）
articles/{id}/final.md（终稿）
     │
     ├──→ grindraft-illustrate → illustrations/{id}/*.png（正文配图）
     ├──→ grindraft-cover      → cover/{id}/*.html + *.png（封面）
     ├──→ grindraft-format     → output.html + output-preview.html（排版 HTML）
     └──→ grindraft-predict    → prediction.md（预测日志，⚠️ immutable）
              │
              ▼（用户发布后）
     grindraft-publish（登记 URL + 时间）
              │
              ▼（T+3 天）
     grindraft-retro ←── 用户手动粘贴数据
              │
              ├──→ rubric_notes.md（评分观察）
              ├──→ style_guide.md（写作 pattern）
              └──→ audience.md（受众画像）
                       │
                       ▼
              grindraft-bump（升级评分公式）
```

## 4. 完整使用流程

### 4.1 初始化

**触发词**："磨稿初始化"、"init"、"首次使用"

**干什么**：
- 创建项目目录结构（`articles/{标题}_{日期}/` 等）
- 写入 7 个骨架文件（rubric_notes.md、style_guide.md、WORKFLOW.md 等）
- 配置你的参数（粉丝数、频率、AI 参与度）
- 判定模式：cold-start（前 5 篇简化）还是 calibration（完整预测）

**初始化后你会看到**：

```
✅ 磨稿初始化完成！

模式：cold-start（剩余 5 篇简化预测）
Rubric：公众号长文 7 维 v0 等权
默认 AI 参与度：high

📂 已创建项目骨架：
  articles/{标题}_{日期}/     — 每篇文章一个文件夹（含 draft/final/封面/配图/排版/预测/复盘）

现在想做什么？
- 抓热点 → "抓热点"
- 聊选题 → "找选题"
- 看状态 → "状态"
```

---

### 4.2 找热点

**触发词**："抓热点"、"今天有什么可写的"、"AI 圈有什么"

**干什么**：
- 从 aihot.virxact.com 拉最近 24 小时 AI 圈精选资讯
- 去重（已写过的不推、6 个月内拒绝过的也不推）
- 用你的评分系统粗打分
- 展示给你，你选哪些加入选题池

**你会看到**：

```
🔥 抓热点完成。aihot 拉到 28 条精选。

去重后剩 19 条。粗打分后 5 条 composite ≥ 5.0：

| # | 标题 | 方向 | composite | 理由 |
|---|---|---|---|---|
| 1 | Claude Code 更新 | AI 产品 | 6.2 | HK=3 UT=3 DT=3 |
| 2 | 阿里云入选 Omdia | 行业趋势 | 5.0 | DT=4 讨论空间大 |
| 3 | ... | ... | ... | ... |

哪些加入选题池？
```

**你不用写代码**。说 "all" 全部加入，或 "1, 3" 选两条。

---

### 4.3 找选题

**触发词**："找选题"、"聊选题"、"我不知道写什么"

**干什么**：
- **不是 AI dump 15 个候选让你挑**。而是围绕你的真实经历深挖
- 你告诉 AI 一件事（"我最近发现..."），AI 问几个问题帮你收敛
- 最终产出：一个 50 字的立意 + 结构骨架

**典型对话**：

```
你: 找选题 我最近发现 AI 写代码让我变懒了

AI: 是什么具体瞬间让你有这种感觉的？

你: 昨天我想写一个脚本，下意识就想让 Claude 先写，
    然后发现我连怎么起头都忘了

AI: 这里面最让你不安的是哪个点？
    是"技能退化了"还是"依赖太深了"还是
    "AI 写的代码我不一定看得懂了"？

你: 第三个。上周有段代码跑通了但我不知道为什么跑通。

AI: 好的。我感觉这个角度能做：

    立意：我们正在从"写代码的人"变成"审代码的人"
    ——而大多数人还没准备好当审代码的人。

    走法：
    - 原型：现象解读型
    - 开头：你那个"跑通了但不知道为什么"的例子
    - 主体：
      1. 现象：程序员正在从 author 变成 reviewer
      2. 分析：reviewer 需要比 author 更高的能力
      3. 深挖：教育和工作流程都没跟上这个转变
    - 收尾：不是反对 AI，是提醒自己别丢掉"理解"的能力

    要不要我按这个写初稿？
```

### 4.4 写初稿

**触发词**："写文章"、"出稿"

**三种参与度**：

| 模式 | 你给什么 | AI 做什么 | 适用场景 |
|---|---|---|---|
| **low** | 完整文章文本 | 润色节奏、补类比、修逻辑 | 你自己写了全文 |
| **medium** | 大纲 + 要点 + 经历片段 | 按大纲扩写，融入你的经历 | 你有清晰想法 |
| **high** | 选题 + 角度（从 seed 产出） | 自规划结构、找素材、写全文 | "给我一版，我改" |

**写稿时会做的**：

1. 读 `style_guide.md` ——了解你的风格偏好
2. 读 `rubric_notes.md` ——知道什么是"好文章"的标准
3. 读 `audience.md`（如果有）——写的时候问"读者想看这个吗"
4. 确定文章原型（解读型/工具型/体验型/方法论型/观点型）
5. 输出结构计划 → 你确认 → 写全文
6. 自检（开头是否从场景切入？有没有套话？收尾是否喊口号？）
7. 落盘到 `articles/{标题}_{日期}/draft.md`，标注 "⚠️ AI 初稿——你需要改写"

### 4.5 去 AI 味

**触发词**："去 AI 味"、"humanize"、"修一下"

**四层质检**（从硬到软）：

```
L1 硬规则扫描（机械扫描——绝对不能有）
  ├─ 禁用词："首先其次最后""综上所述""值得注意的是"...
  ├─ 结构套话：连续 bullet ≥4 条、结尾喊口号
  └─ 用户个人禁区（从 style_guide.md 读取）

L2 风格一致（模式匹配——读起来像话吗）
  ├─ 开头是否从具体场景切入？
  ├─ 节奏是否长短句交替？
  ├─ 是否用了口语化过渡词？（≥5 个不同的）
  ├─ 是否有自嘲或承认不确定？
  └─ 结尾是否避免了喊口号？

L3 内容质量（深度审查——有说服力吗）
  ├─ 每个观点是否有具体细节支撑？
  ├─ 知识点是"聊天式掏出"还是"教科书介绍"？
  ├─ 是否有对对立面的理解？
  └─ 文章原型专项检查

L4 活人感终审（最终判断——像人写的吗）
  ├─ 温度感：体感记忆 vs 知识描述
  ├─ 独特性：换个人+AI 能写出一样的吗？
  ├─ 姿态：是聊天还是教书？
  └─ 心流：有没有地方注意力断了？
```

**你会看到**：

```
## 去 AI 味 — 质检报告

L1 硬规则 ✅
- 禁用词：2 处命中（"首先"→已修，"值得注意的是"→已删）

L2 风格一致 ❌
- 开头：✅
- 节奏：❌ 第 3-5 段句式长度全在 25-30 字，缺断裂感
- 口语化：✅（用了 7 个不同口语词组）
- 结尾：✅

L3 内容质量 ✅

L4 活人感 ❌
- 温度感：第 8 段"我感到非常震撼"→改成体感记忆
- 心流：第 12 段转折生硬，注意力断了

修复优先级：
1. 第 8 段改情绪表达
2. 第 3-5 段打破句式
3. 第 12 段加过渡句
```

**你的个人禁区自动沉淀**：每次你改掉 AI 写的某个表达，说"这个我不喜欢"——磨稿记到 `style_guide.md`。下次去 AI 味时自动叠加到 L1 扫描。

### 4.6 设计封面

**触发词**："设计封面"、"封面"、"生成封面"

**干什么**：
- 读你的终稿，做**三维分析**：文章情绪（冷峻/温暖/激昂/轻松/严肃）× 内容领域（科技/商业/生活/教育/娱乐/人文）× IP 一致性
- 从 **40 套模板**中推荐最匹配的 3 个（带色板和推荐理由）
- 提取封面文案：标题（≤15 字）+ 一句话描述 + 标签
- 生成双图预览 HTML（2.35:1 公众号首图 + 1:1 圆角小图，左右并排）

**你会看到**：

```
🎨 文章分析：情绪「冷峻」× 领域「科技」
   推荐模板：
   ① 赛博霓虹     — 暗底+青品红辉光，适合技术深度文 ✅ 强烈推荐
   ② 工程蓝图     — 网格+青色调，适合教程/开发者内容
   ③ 全息未来     — 虹彩渐变，适合前沿科技话题

   封面文案：
   标题：AI 写代码让我变懒了
   描述：从"写代码的人"变成"审代码的人"
   标签：#AI编程 #程序员

   预览：articles/xxx/cover/preview.html
   （双击预览 HTML 看效果）

   下一步：确认 → 调整文案 → 换模板 → 出 PNG
```

**支持改稿循环**（最多 5 轮）：
- "封面文案换成 XX" → 更新
- "换成赛博霓虹" → 换模板
- "配色调暖一点" → 调色板
- "出 PNG" → 需要 Node.js + puppeteer + canvas

### 4.7 排版

**触发词**："排版"、"转 HTML"、"公众号排版"

**干什么**：
- 读你的终稿 `articles/{标题}_{日期}/final.md`
- 如尚未做封面 → 自动提醒："要不要先设计封面？封面图会嵌入排版 header"
- 按你的公众号排版规范转换成 HTML（section 嵌套、内联样式、flex 布局）
- 外链自动转底部引用
- 输出双版文件到文章目录

**你会看到**：

```
✅ 排版完成：articles/{标题}_{日期}/
  ├── output.html          ← 清洁版，直接粘贴公众号后台
  └── output-preview.html  ← 预览版，双击看手机效果

主题：Vercel 极客 | 字数：4200 | 外链引用：3 条（已转底部引用）

下一步：
  - "启动预测" → 写预测日志
  - 如未做封面 → "设计封面"
```

### 4.8 盲预测

**触发词**："启动预测"

**干什么（cold-start 模式）**：
- 给你的终稿打 7 维分（每维 1-5 + 理由）
- 给一个方向押注："比上一篇明显好 / 差不多 / 可能不如"
- 写入 `articles/{标题}_{日期}/prediction.md`

**干什么（calibration 模式，前 5 篇后解锁）**：
- 7 维分 → composite → bucket 押注（S/A/B/C/D/E）
- 所有 bucket 的概率分布（"B: 50%，C: 30%，A: 15%..."）
- 锚点对比（和历史上 composite 接近的文章比）
- 反事实场景（"如果 >A bucket → 说明 HK 被低估了"）
- 关键校准假设（"本篇 vs 上一篇押 1.5x"）

**⚠️ 预测一旦写入文件即永久锁定。** 你发布了、看了数据后，不能再改。如果真需要重做，走 `_redo.md` 路径（标注"这不是盲预测"）。

**你会看到**：

```
📊 我的预测草稿（确认后即锁定）：

7 维分（v0 等权）：
| HK=4 | NR=4 | QA=3 | UT=3 | DT=4 | EP=2 | SC=3 |
→ composite ≈ 7.7

方向押注：比上一篇明显好（上一篇 composite 6.2）
锚点：N/A（第 1 篇预测）

回 "ok" 落盘，或指出哪个维度不对。
```

### 4.9 发布登记

**触发词**："已发布"、"发出去了"

**干什么**：
- 在预测文件 header 补上发布时间和 URL
- 更新 `.grindraft-state.json`（清 in_progress，加到待复盘列表）
- 提醒你盲度窗口已关闭——从此看到的数据不能用来"补预测"
- 提醒 T+3 天后该复盘

**你会看到**：

```
✅ 发布登记完成

⚠️ 从此刻起，你看到任何阅读/分享/留言数据
   都会破坏盲度声明的诚信。

📅 计划复盘：T+3 天，约 2026-01-18
   到时间说："复盘 articles/{标题}_{日期}"
```

### 4.10 复盘

**触发词**："复盘"

**时间**：发布后至少 3 天（让数据沉淀）

**干什么**：
1. AI 让你打开公众号后台，粘贴数据：
   ```
   阅读量：_______
   分享数：_______
   在看数：_______
   留言数：_______
   收藏数：_______
   阅读完成率：______%
   ```
2. AI 让你粘贴 Top 20 留言（带赞数）
3. AI 做三件事：
   - **对比预测**："HK=4 被验证了，打开率高于均值" / "NR=3 低估了，完读率 72%"
   - **提炼观察**：什么评分规则要修正？什么写作 pattern 有效？
   - **检测升级**：连续 N 次偏差同向？→ 提议升级 rubric

**你会看到**：

```
📊 复盘完成：articles/2026-01-15_a1b2c3d4_审代码的人/prediction.md

实绩 vs 预测：
  阅读量：5,200 — 落在 B bucket（预测也是 B）✅
  分享率：4.2% — 高于均值 1.8 倍
  完读率：68% — NR=4 被验证 ✅
  留言：32 条 — DT=4 被验证，但质量低于预期：60% 是"好文"类低信号

哪些预测被验证 ✅ / 推翻 ❌：
  ✅ HK=4：打开率高于同粉均值 1.3 倍
  ✅ NR=4：完读率 68%，叙事牵引力判断准确
  ❌ EP=2 可能高估了——分享率好但留言里几乎没有情绪反馈

需要写进 rubric_notes.md 的新观察：
  1. DT=4 但留言质量低 → 下次区分"留言数量"和"留言深度"
  2. EP 的判断需要更严格的锚点

需要写进 style_guide.md 的新 pattern 候选：
  1. 你砍掉了 AI 写的第 4-5 段（数据罗列），改成了个人故事
     → 完读率在第 4 段之后没掉，说明改得对
     → 候选 Pattern：数据罗列不如个人故事（≥1 样本待验证）
```

### 4.11 状态看板

**触发词**："状态"、"看板"

**任何时候可调，无副作用**。展示：

```
📊 模式：cold-start（3/5 篇剩余）
📝 选题池：🟢 充裕（5 条待写）
📈 待复盘：2 篇（其中 1 篇可复盘）
🎯 校准：连续 2 次低估 → 暂不建议 bump（样本不够）
🧹 维护：style_guide.md 最后更新于 10 天前
```

### 4.12 受众画像

**触发词**："我的读者是谁"、"persona"

**前置条件**：≥3 篇有留言数据的复盘

**干什么**：
- 从所有历史留言中聚类读者特征
- 输出画像：年龄/职业推断、共同痛点、阅读动机、转发动机
- 和 rubric 交叉检验——"rubric 说 NR=3 但读者留言说'一口气读完'→ NR 可能被低估了"

**污染隔离**：`audience.md` 含实绩信号，**blind sub-agent 永远不读**。persona 只影响选题方向，不影响评分。

### 4.13 升级评分系统

**触发词**："升级 rubric"、"bump"

**前置条件**：≥5 篇复盘数据 + 连续偏差信号

**干什么**：
- 识别升级方向（调权重 / 调 bucket 边界 / 新增维度 / 删除维度）
- **全量重打**：用新公式对所有历史文章重新评分，新排序与实际排序 ≥4/5 一致 → 通过
- **跨模型审核**：把新公式 + 5 篇样本发给独立模型打分，差异 ≥2 的维度不过 2 个 → 通过
- **Cleanup**：删掉已被吸收的观察（不在 markdown 里留考古层）

## 5. 核心概念详解

### 5.1 7 维评分体系

磨稿用 7 个维度给每篇文章打分：

| # | 维度 | 简称 | 测什么 | 公众号后台验证 |
|---|---|---|---|---|
| 1 | **标题钩力** | HK | 标题能不能让人想点进来 | 阅读量 |
| 2 | **叙事牵引力** | NR | 读了三段后有没有一路看到底的欲望 | 阅读完成率 |
| 3 | **金句锚点** | QA | 有没有值得划线的句子 | 划线数 / 在看数 |
| 4 | **实用密度** | UT | 有没有"以后用得上"的东西 | 收藏率 |
| 5 | **思辨激发度** | DT | 有没有让人想留言讨论的问题 | 留言率 + 留言质量 |
| 6 | **情绪峰值** | EP | 有没有至少一个"卧槽"或鼻酸的瞬间 | 分享率 |
| 7 | **结构闭环** | SC | 开头埋的线结尾有没有收回来 | 读完后的完整感 |

每维 1-5 分。初始权重（等你复盘校准后会变）：

```
composite = (HK×2.0 + NR×2.0 + QA×1.5 + UT×1.5 + DT×1.0 + EP×1.0 + SC×0.5) / 9.5 × 5
```

Composite 映射到阅读量预测：

| Bucket | 阅读量范围 |
|---|---|
| S | >50,000 |
| A | 10,000-50,000 |
| B | 3,000-10,000 |
| C | 1,000-3,000 |
| D | 300-1,000 |
| E | <300 |

> Bucket 边界随你的粉丝数自动调整。4000 粉时各项 +40%。

### 5.2 三条铁律

**铁律 1：盲预测不可改**

预测在发布前写。一旦写了，那个 `## 预测` 段永久锁定。发布后看到了数据 → 不能回去改。如果真需要重做（比如稿子版本搞错了），走 `_redo.md` 路径，原文件保留。

**铁律 2：升级必须全量重打**

改评分公式时，必须用新公式重新评估所有历史文章。新排序 vs 真实阅读量排序，≥4/5 一致才能通过。不能"我觉得 NR 应该 ×3"——要数据证明。

**铁律 3：规则本不是博物馆**

被推翻的假设 → 删。被吸收到公式里的观察 → 删。不要在 markdown 里留"我曾经以为 X 但后来发现 Y"。git 历史是档案，不是 markdown 文件。

### 5.3 Cold-start vs Calibration

| | Cold-start | Calibration |
|---|---|---|
| **适用** | 前 5 篇文章 | 第 6 篇开始 |
| **打分** | 7 维等权（各 ×1.0） | 7 维加权 |
| **预测内容** | 7 维分 + 方向押注（"比上一篇好/差"） | 完整 7 组件（bucket + 概率 + 锚点 + 反事实） |
| **为什么** | 没数据时精确数字是 false precision | 有数据支撑后解锁完整分析 |

第 5 篇复盘后，`grindraft-status` 会主动提示"完整预测模式已解锁"。

### 5.4 AI 参与度 (Autonomy)

| autonomy | 你给 | AI 做 | 什么时候用 |
|---|---|---|---|
| **low** | 完整文章 | 润色节奏、补类比、去套话 | 你喜欢自己写 |
| **medium** | 大纲 + 要点 + 经历 | 按大纲扩写、融入经历 | 你有清晰想法 |
| **high** | 选题 + 角度 | 出初稿，你来改 | 你想快速出稿 |

初始化时设默认，每次写之前可以临时改："写文章 --autonomy medium"。

### 5.5 风格指南的自我进化

`style_guide.md` 是你写作风格的"基因库"。初识时是空的。每次你改 AI 的稿子，磨稿观察"AI 写了什么 → 你改成了什么"，自动沉淀：

```
第 1 篇：你删了 AI 写的长排比 → 记"避免排比句"
第 3 篇：你把 AI 的 5 段论砍成 3 段 → 记"偏好三段结构"
第 5 篇：你重写了所有引言 → 记"引言要从个人行为切入"
...
第 10 篇：磨稿说"我发现你 80% 的开头是'昨天/今天 + 具体行为'"
```

这些 pattern 叠加到去 AI 味的 L1 扫描和 L2 风格检查里。**你用多久，它学多久。**

## 7. 常见问题

**Q：我不确定该设什么 autonomy？**

A：刚开始用磨稿建议 `high`。让 AI 先出初稿，你感受一下"我需要改多少"。如果发现 AI 写的方向经常不对 → 降到 `medium`（你给大纲）。如果你发现自己写更顺手 → 降到 `low`。可以随时换。

**Q：7 个维度我都能自己调吗？**

A：初始权重是 AI 给的先验猜测。当你有 ≥5 篇复盘数据后，可以用"升级 rubric"调整。不能手动改权重——必须走 bump 流程（全量重打 + 跨模型审），确保改动是被数据验证过的。

**Q：预测错了怎么办？**

A：预测精准度本身不重要——重要的是"复盘时搞清楚为什么错"。一次预测错误 = rubric 的一次进化机会。连续错同一方向 3 次 = 系统建议升级公式。**预测的价值不在于准，在于帮你发现自己对内容判断的盲区。**

**Q：我可以不写预测直接发吗？**

A：可以。但不写预测就不能复盘。不复盘的写作就是"凭感觉"——磨稿的核心价值就没用了。

**Q：style_guide.md 是空的，有用吗？**

A：前几篇文章它确实是空的——没关系。磨稿的去 AI 味有通用规则（L1-L4），不依赖你的个人禁区。style_guide.md 是"锦上添花"——当你改了 5-10 篇后，它会让你省很多事。

**Q：磨稿和直接让 Claude 写一篇有什么区别？**

A：直接让 Claude 写 = 一次性。磨稿 = 每篇都在训练你的写作系统。差别在第 10 篇开始显现——rubric 知道你的文章什么维度重要，style_guide 知道你不喜欢什么表达，persona 知道你的读者为什么转发。第 10 篇时磨稿出的初稿，比第一次用"直接写"精准得多。

**Q：从零开始（cold-start）需要多久进入 calibration？**

A：发 5 篇 + 每篇 T+3 天后复盘 = 最短约 3-4 周。如果你周 2 篇，大约一个月后解锁完整模式。

**Q：我在别的项目里能复用 rubic 和 style_guide 吗？**

A：`.grindraft-state.json`、`rubric_notes.md`、`style_guide.md`、`audience.md` 可以复制到新项目——它们代表你在**这个账号**上的积累。但要注意：不同公众号的读者不同，rubric 可能需要重新校准。
