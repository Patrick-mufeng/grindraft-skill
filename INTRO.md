# 磨稿 / Grindraft

> Claude Code 插件 · 16 个 skill · 把公众号写作变成可进化的校准系统

---

## 一句话

**不是帮你写文章，是帮你建立一套"每写一篇，下一篇就更准"的写作系统。**

---

## 它和普通 AI 写作工具有什么区别？

普通 AI 写作：你让它写 → 它写 → 写完了。下次再写，它不记得你上一篇阅读量 3000 还是 300。

磨稿：**每一篇文章都变成数据，数据反过来校准你的写作判断。**

```
写 → 预测 → 发布 → 复盘 → 进化
 ↑                        ↓
 └──── 更准的评分系统 ────┘
```

---

## 怎么做到的？三条铁律

### 1. 盲预测 — 预测写了就不能改

每次发文章之前，AI 先写一份预测："我觉得这篇阅读量会在 3000-10000，因为标题钩力 4 分、情绪峰值 5 分……"一旦写完，**这段预测永久锁定**。发了文章看到真实数据后，不能回头改预测。预测和实际的差距，就是你的评分系统需要进化的地方。

### 2. 升级必须全量重打

当你觉得评分系统不准了（比如每次预测都低估实际阅读量），可以升级。但升级后必须把**所有历史文章用新公式重新打一遍**——新排序和实际排序至少 80% 一致才能通过。不是"我觉得 HK 应该 ×3"，而是用数据证明"×3 确实让历史的排序更接近真实"。

### 3. 规则本不是博物馆

被数据推翻的假设、被吸收到正式规则里的观察——**删掉，不留考古层**。Git history 才是档案。"我曾经以为标题最重要，但其实……"这种句子，删。如果它已经被吸收为公式里 HK×2.0 的那部分。

| 阶段 | Skill | 一句话 |
|------|-------|--------|
| 🔍 发现 | `grindraft-trends` | 抓今日 AI 圈精选资讯，写入选题池 |
| 💡 选题 | `grindraft-seed` | 围绕你的真实经历深挖，收敛到一个角度 |
| ✍️ 写作 | `grindraft-write` | AI 出初稿（3 种参与度，你决定 AI 做多少） |
| 🧹 打磨 | `grindraft-humanize` | 四层去 AI 味自检（L1 硬规则 → L4 活人感） |
| 🎨 封面 | `grindraft-cover` | 读文章自动匹配 40 套模板，生成预览 |
| 📐 排版 | `grindraft-format` | Markdown 一键转公众号 HTML |
| 🎯 校准 | `grindraft-predict` | 7 维盲打分 + bucket 押注（⚠️ 写完不可改） |
| 📊 复盘 | `grindraft-retro` | T+3 天数据回收，逐条验证/推翻预测 |
| ⚙️ 进化 | `grindraft-bump` | Rubric 公式升级（全量重打 + 跨模型审核） |

## 7 维评分体系

AI 的写作判断被拆成 7 个可验证的维度，每个维度对应公众号后台的一个真实数据：

| 维度 | 测什么 | 复盘用哪个数据验证 |
|------|--------|-------------------|
| **HK** 标题钩力 | 标题能不能让人点进来 | 阅读量 |
| **NR** 叙事牵引力 | 有没有一路看到底的欲望 | 阅读完成率 |
| **QA** 金句锚点 | 有没有值得分享的句子 | 划线数 / 在看数 |
| **UT** 实用密度 | 看完有没有"以后用得上" | 收藏率 |
| **DT** 思辨激发度 | 有没有让人想讨论 | 留言率 + 留言质量 |
| **EP** 情绪峰值 | 有没有"卧槽"或鼻子一酸 | 分享率 |
| **SC** 结构闭环 | 开头埋的线结尾有没有收 | 读完后的完整感 |

每维 1-5 分，加权算 composite，映射到阅读量 bucket（S/A/B/C/D/E）。权重不是固定的——**每次复盘后，被验证的维度权重上升，被推翻的维度权重下降。**

## 两种模式

### Cold-start（从零开始）
前 5 篇文章走简化预测——只打 7 维分 + 方向押注（"比上一篇好/差不多/不如"），不做精确数字预测。因为没数据时精确数字是 false precision。第 5 篇复盘后自动解锁完整模式。

### Calibration（有数据积累）
≥5 篇复盘数据 → 完整 7 组件预测：打分 + bucket + 概率分布 + 锚点对比 + 反事实场景 + 关键校准假设。rubric 从 v0 等权进化到 v1 加权，持续随数据进化。

## 安装

```bash
# Claude Code 插件市场
claude plugin install github.com/你的用户名/grindraft-skill

# 或手动复制到项目
cp -r skills/* ~/your-project/.claude/skills/
```

安装后在 Claude Code 中说 **"磨稿初始化"** 即可开始。

## AI 的角色

磨稿的 AI 定位是**写手**，不是导师。它说"我帮你出了一稿，你看哪里需要改"——不是"你应该这样写"。

**AI 擅长的**：找背景知识、找类比和比喻、按大纲扩写、检查逻辑漏洞和节奏问题。

**AI 做不到的**：一手观察和真实经历、核心创意角度、情绪的真实表达——这些必须你来。
